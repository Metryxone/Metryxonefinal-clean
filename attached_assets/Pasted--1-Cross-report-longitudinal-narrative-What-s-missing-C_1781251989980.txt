
1 · Cross-report longitudinal narrative
What's missing: CAPADEX sessions, competency scores, and MEI snapshots are all timestamped but never shown as a single development timeline. A user completed 3 CAPADEX sessions and 5 competency assessments — that story is invisible.

Fix: Surface WC-L5 Memory Intelligence (already composed from L0–L4) as a "Your development over time" thread that IntelligenceLayers reads for every report. One timeline, five entry points, shared data.

2 · Decision orchestrator surface (WC-7B exists, nobody reads it)
What's missing: The WC-7B activation decision orchestrator composes WC-3 L1/L2/L3 and writes a "next best action" — but none of the 5 reports calls it. Each report currently recommends in isolation.

Fix: Add a sixth tab Next Steps to IntelligenceLayers that calls GET /api/capadex/session/:id/decision (already exists) and surfaces a single prioritised action list. One decision layer, all 5 reports benefit.

3 · Behavioral signal → competency priority modulation
What's missing: IntelligenceLayers fetches CIE gaps independently of CAPADEX behavioural data. But the career-behavior-adapter already maps CAPADEX concern signals to competency modifiers — those modifiers aren't being applied to re-rank the gap list.

Fix: When sessionId is present, pass the CAPADEX behaviour context into the gap prioritisation rendering so "Analytical Reasoning" rises higher if the behavioural spine shows a matching concern pattern. No new engine — just a client-side re-rank using already-fetched data.

4 · Data completeness transparency layer
What's missing: Empty tabs give a generic "complete an assessment" message but don't say why they're empty or how close the user is to activating them. A cold-start user can't distinguish "you need one more session" from "this feature needs your target role set."

Fix: Each empty-state should emit a specific readiness signal (e.g. trends: need 1 more session, outcomes: set a target role, interventions: no critical gaps found). IntelligenceLayers already has the summary payload to derive this — it's a rendering change, not an API change.

5 · Strength signal — currently absent from IntelligenceLayers
What's missing: The Patterns tab shows gaps and behavioural concern patterns, but strengths (from CSI positive_factors / positive longitudinal growth) are never surfaced. Reports read as deficit-heavy.

Fix: Fetch GET /api/capadex/session/:id/patterns already returns positive_factors when available. Add a "Strengths" subsection to the Patterns tab alongside the gap patterns. No new data — it's already in the response.
