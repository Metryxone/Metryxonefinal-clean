MISSION

Create the permanent Frontend Architecture Constitution for CAPADEX.

This document defines how every frontend enhancement must be designed,
implemented, reviewed and maintained.

The objective is to evolve the existing frontend into a world-class enterprise
architecture WITHOUT rebuilding it.

Repository remains the source of truth.

==============================================================================

PRIMARY OBJECTIVES

Standardize frontend architecture.

Protect existing implementation.

Maximize reuse.

Improve maintainability.

Improve scalability.

Reduce technical debt.

Increase consistency.

Prepare the platform for long-term evolution.

==============================================================================

MANDATORY RULES

Before changing any frontend code

Search the repository.

Locate the existing implementation.

Understand why it exists.

Document current behaviour.

Enhance existing implementation.

Never create duplicate components.

Never create duplicate pages.

Never create duplicate layouts.

Never create duplicate hooks.

Never create duplicate stores.

Never create duplicate contexts.

Never create duplicate UI primitives.
