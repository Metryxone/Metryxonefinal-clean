# CAPADEX 2.0 — Engineering Execution Specification

# PHASE 1.4 — UX CONSTITUTION (Experience Bible)

> **Execution Mode:** ENHANCEMENT ONLY
>
> **DO NOT IMPLEMENT UI CHANGES**
>
> **DO NOT REBUILD THE FRONTEND**
>
> **DO NOT MODIFY BUSINESS LOGIC**
>
> This phase establishes the permanent UX Constitution that every future enhancement must follow.

---

# MISSION

Create the permanent User Experience Constitution for CAPADEX.

CAPADEX is not a collection of pages.

It is a continuous Behaviour Intelligence experience.

Every screen, interaction, recommendation, assessment, journey, report and AI conversation must feel like one unified product.

This constitution defines that experience.

---

# PRIMARY OBJECTIVES

Establish one UX philosophy.

Standardize every user interaction.

Define world-class experience principles.

Transform CAPADEX from a traditional assessment platform into an engaging Behaviour Intelligence platform.

Ensure every future enhancement improves usability.

---

# MANDATORY RULES

Never redesign a screen without understanding its existing purpose.

Never duplicate workflows.

Never introduce inconsistent navigation.

Never create different interaction patterns for similar features.

Reuse existing components wherever possible.

Enhance before replacing.

Consistency always wins over novelty.
