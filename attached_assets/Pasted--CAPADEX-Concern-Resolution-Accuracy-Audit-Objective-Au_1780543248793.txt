# CAPADEX Concern Resolution Accuracy Audit

## Objective

Audit the entire Concern Resolution layer before implementing any fixes.

Investigation only.

Do NOT modify code.

Do NOT update mappings.

Do NOT change scoring.

Measure current system accuracy.

---

## Context

Recent architecture audit revealed:

1. User intent may drift during resolveMasterConcernIdFromText()
2. "Frequently Changes Career Goals" does not resolve to a dedicated concern
3. Keyword/stem matching may route users into adjacent concerns
4. Downstream question quality depends on concern resolution quality

We need to quantify the problem before implementing changes.

# PHASE 3 – Drift Analysis

Identify:

### Exact Match

User intent resolves to intended concern.

### Near Match

User intent resolves to semantically adjacent concern.

### Drift Match

User intent resolves to a different construct.

### Failure

No concern resolved.

Generate:

Top 500 Drift Matches

Top 200 Failures

Top 100 Ambiguous Intents
