# Objective
Refactor the pre-questions profile brief layout ("image_c7f7f4.png") to display highly specific, contextual patterns derived from the resolved concern's metadata (cluster, domain, labels) instead of rendering static, hardcoded placeholders.

---

## Technical Specification 1: Concern-Derived Pattern Hydration
File: `backend/routes/capadex-concern-intelligence.ts` (Inside the initial `analyzeConcern` or pre-screen return payload)

1. When a concern (like 'Weak Ability to Sustain Focus During Long Work Hours') is resolved, intercept the payload before it leaves the server.
2. Build a mapping dictionary that translates master concern clusters or domain tags into distinct, high-fidelity behavioral patterns:
   - **If Domain is ACADEMIC_COGNITIVE & Persona is WORKING_PROFESSIONAL**:
     - `P1`: "Context-Switching Attenuation" (Instead of Direction Ambiguity)
     - `P2`: "Deep-Work Boundary Erosion" (Instead of Identity-Role Mismatch)
     - `P3`: "Cognitive Backlog Overload" (Instead of Decision Paralysis)
     - `tags`: ["Focus Fragment", "Task Fatigue", "Attention Drag"]
   - **If Domain is ACADEMIC_COGNITIVE & Persona is STUDENT**:
     - `P1`: "Exam-Curriculum Calibration Strain"
     - `P2`: "Study-Block Disengagement"
     - `P3`: "Retention Evaluation Anxiety"
     - `tags`: ["Anxiety", "Backlog", "Procrastination"]
3. Surface this contextual array to the frontend inside an object key named `preliminary_patterns`.

---

## Technical Specification 2: Frontend Progressive Hydration Wiring
File: `frontend/src/components/assessment/phases/CapadexReportPhase.tsx` (or your local component rendering the Profile Brief preview)

1. Locate the block rendering the "WHAT'S ALREADY VISIBLE ABOUT YOU" card container visible in "image_c7f7f4.png".
2. Bind the text properties of the P1, P2, and P3 cards directly to the incoming `preliminary_patterns` payload values sent by the resolved concern metadata.
3. **Add Progressive Micro-Hydration**: As the user answers the clarity questions within the modal, append those localized signals directly to the preview state. For example, if a user selects a highly stressful option, immediately swap out the default background tag (`Uncertainty`) for an active interaction tag (`Acute Evaluation Stress`).

---

## Verification Criteria
- Launching an assessment for a Working Professional typing "focus at work" instantly replaces the generic placeholders in "image_c7f7f4.png" with contextual workplace terms like "Context-Switching Attenuation".
- Changing the persona or concern text dynamically updates the preview text categories before a single clarity question is answered.
- The UI retains its seamless layout structure with no rendering layout jumps or broken text wrappers.
