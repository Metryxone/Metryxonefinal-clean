## Part 2: Core Mathematical & Behavioral Logic

### 2.1 The Profile Completeness Ceiling Rule
The platform enforces a mathematical progress ceiling to prevent advanced tracking on incomplete profiles[cite: 1]. Profile completeness (0-100%) scales the maximum proficiency level across all 25 competency domains using this precise formula[cite: 1]:
$$\text{Proficiency Level Ceiling} = \min\left(5, 1 + \left\lfloor \frac{\text{Profile Completeness Percentage}}{22} \right\rfloor\right)$$
- Implement this exact ceiling logic[cite: 1]. If a user’s profile is only 44% complete, their skill scores are automatically capped at Level 3[cite: 1].
- If this ceiling is active, inject a visible warning card banner on the main Dashboard to notify the user of why their scores are capped and how to unlock higher competency ranges[cite: 1].

### 2.2 Rule-Based Competency Boosters
Build an automated file parser (`careerIntelligence.ts`) that scans a user's background history to inject baseline competency scores prior to testing[cite: 1]:
- Skill Counts: $\ge 3$ skills sets `programming` to Level 2; $\ge 6$ skills scales it to Level 3; $\ge 10$ skills overrides it to Level 4[cite: 1].
- Project Count: $\ge 3$ projects sets `programming` to Level 3[cite: 1].
- Experience Tenure: $\ge 4$ years boosts `people-mgmt` to Level 2 and `stakeholder-mgmt` to Level 3[cite: 1]. At $\ge 7$ years, `strategy` and `mentoring` lock at Level 3[cite: 1].

### 2.3 Master Workspace Orchestrator (`useCareerBrain`)
Implement a global state context provider (`CareerBrainProvider`) and hook (`useCareerBrain`) to sync client-side persistence keys across all 19 views[cite: 1]. It must handle:
- Key Synchronization: `metryx_token`, `mx-career-target-role`, `mx-career-visibility-open`, and `mx-career-idp-progress`[cite: 1].
- Live Evaluation: Instantly trigger `computeFitment()`, `buildIDP()` (sorting up to 7 items by highest `eiLift / hours` ROI), and `computeVisibility()` whenever the profile updates, ensuring no state is lost when tabs change[cite: 1].
