First: the LBI / "EI" conceptual error you flagged — confirmed
You are correct. "LBI Assessment Results" is the wrong data source label for this screen, and "EI Score" is mislabeled too. Here's the evidence from the real code:

LBI = Learning Behavioural Intelligence — it measures how a person learns: Consistency, Persistence, Attention, Adaptability, Velocity, plus a "learning style." It was built for students / career-seekers (CAPADEX). It is not a workforce performance/retention engine.
"EI" on this screen = Employability Index (MetryxOne's 0–99 composite), not Emotional Intelligence. Yet the card sub-label literally says "Behavioural index" (:6429). That's a third, different concept. Three names, one number — muddled.
The real backend that should power this screen disagrees with the screen. GET /api/employer/eios/employees/:id/intelligence builds an employee's Composite Index from three sources:
Behavioural ← wcl0_user_intelligence (motivation / adaptability / engagement / risk)
Cognitive ← capadex_reports.overall_score
Future Readiness ← frp_user_readiness.fri_score
It never reads the lbi_* tables at all. So LBI is not the primary feed — it's at most a minor indirect contributor (~one sub-input). The honest channel name would be "CAPADEX behavioural + cognitive assessment + FRP readiness," not "LBI Assessment Results."
