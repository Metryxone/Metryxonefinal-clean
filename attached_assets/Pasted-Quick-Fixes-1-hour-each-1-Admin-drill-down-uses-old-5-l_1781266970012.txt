Quick Fixes (< 1 hour each)
1. Admin drill-down uses old 5-layer endpoint
The "User Drill-down" tab in the admin panel calls /api/admin/lbi/intelligence/:email (profile + 4 LBI layers). It should call the new /api/lbi/intelligence?adminEmail=X which returns all 7 layers including WCL2 forecast, WCL3 outcomes, comparative, and causal recs. One route swap.

2. Forecast tab has two identical icons
Both "Learning Trends" and "Forecast" tabs show <TrendingUp size={15} />. Forecast should use something distinct (e.g. <LineChart> or <Sparkles>).

Medium Enhancements (1–3 days each)
3. LBI Learning Style → Intervention mapping
Interventions are currently ranked by target_dimension + effort_level only. The learning_style (reflective / persistent / exploratory / impulsive / disengaged) is computed but not used as a ranking signal. A learning_style multiplier on the recommendation engine would make interventions genuinely personalized — a disengaged learner needs shorter, low-effort tasks; a reflective learner benefits from longer analytical exercises.

4. LBI → Career Passport sync
The Career Passport (syncPassportFromPlatform) bridges CAPADEX / FRP / competency data into the cp_* tables — but LBI is absent. LBI overall score, learning style, and dominant risk should be a Passport section. That's one syncLbiToPassport() function + a section in SECTION_CONFIG.

5. LBI chain trigger on non-CAPADEX events
The chain (calculateAndPersistLBI → trends → risks → recs → longitudinal) only fires on CAPADEX session completion. It doesn't fire when:

A competency assessment is completed (career builder)
A Career OS profile is updated
An FRP snapshot is taken
Adding the trigger to those events would increase LBI coverage without waiting for users to retake CAPADEX.

6. WCL3 outcomes shape verification + defensive mapping
The /api/lbi/outcomes route passes getSessionOutcomes() output directly. The frontend accesses outcomesData?.outcomes?.outcomes (nested). This works only if OutcomeSummary has an outcomes array inside. Worth a smoke-test with a real session to confirm the nesting is correct — if off, the Outcomes tab silently shows empty.

Strategic Enhancements (1–2 weeks each)
7. LBI ↔ FRP cross-platform bridge
LBI and Future Readiness Platform measure overlapping dimensions:

LBI	FRP signal
adaptability_score	adaptability (20%)
engagement_score	learning_velocity (15%)
persistence_score	skill_durability proxy
Building a bidirectional enrichment — LBI behavioral signals feeding into FRP confidence, FRP market alignment feeding into LBI risk scoring — would make both scores more robust and require fewer sessions to reach reliable confidence levels.

8. LBI stakeholder report (RF template)
Currently the report archives a raw summary to rf_generated_reports. The Report Factory has a proper template + section system (rf_templates, rf_report_sections) with multi-stakeholder rendering (parent / counselor / employer views). A proper LBI RF template with 4 stakeholder variants (similar to WC-3's Dynamic Report Intelligence) would make the LBI report something that can be emailed, white-labeled, and sold — rather than just an in-app JSON.

9. LBI Enterprise cohort analytics
Aggregate LBI scores by cohort/team/industry for HR dashboard. The Enterprise Analytics module has fact tables but LBI data isn't feeding in. This unlocks the B2B product angle: an employer can see the aggregate learning style distribution of their team and which risk profiles are most prevalent.

10. LBI longitudinal trajectory when < 2 snapshots
Currently the trajectory shows stable/low because ≥2 snapshots are needed. The WCL2 forecast already projects d30/d60/d90 — that data could be used to show a projected trajectory on the Longitudinal tab even before a second session exists, with a clear "Projected (1 session)" label. Honest and more useful than "stable/low."
