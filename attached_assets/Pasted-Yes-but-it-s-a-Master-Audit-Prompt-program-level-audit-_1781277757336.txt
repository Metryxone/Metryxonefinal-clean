### Mandatory Rules

* No assumptions

* No architectural claims without evidence

* Verify DB row counts

* Verify frontend consumers

* Verify backend producers

* Verify route registration

* Verify actual runtime paths

* Separate:

  * Built
  * Activated
  * Data-backed
  * Production-ready

* Identify:

  * dead code
  * orphan tables
  * disconnected intelligence
  * duplicated logic
  * mock data
  * placeholder implementations

**STOP ONLY if a fundamental architectural conflict is discovered.**

This is the kind of audit prompt I would use if your goal is to produce an Employer Portal audit at the same rigor as the EI, LBI, Career Builder, PX-1, and PX-2 audits.
