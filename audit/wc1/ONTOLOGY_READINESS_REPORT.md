**Three findings drive the score down:**

1. **The signal layer is the systemic gap.** Only **25 of 328 bridge tags (7.6%)** carry any atomic signals. The **15,972** tagged atomic signals span just **28 distinct bridge tags**; of the 328 concern bridge tags only **25 overlap** (have both concerns and signals), and just **6,761 of the 15,972** atomic signals (≈42%) fall under a tag that also carries concerns — the rest sit under a handful of signal-heavy, concern-light tags. ~300 concern tags have **zero diagnostic signal grounding**. This is the single biggest ontology-completeness risk.
2. **No marketed construct is "first-class."** Not one of the 10 marketed constructs has the full stack of *dedicated bridge tag + dedicated atomic signals + clarity questions + behaviour + archetype + intervention/recommendation*. GREEN is therefore unreachable today — every marketed construct is at best partially wired.
3. **Interventions & recommendations live in a separate namespace.** `recommendation_library.anchor_construct` (33 values) and `intervention_library.construct_key` (38 values) use a compact construct vocabulary (ANXIETY, CAREER_CLARITY, COMMUNICATION…) that is **not the 328 bridge tags**. Only **113 of 328 tags (34.5%)** *lexically* map to a recommendation construct; the real binding is **runtime-only** (no stored bridge-tag ↔ construct edge exists). The 34.5% figure is therefore a **lexical proxy**, not a verified edge (see §7).

## 7. How coverage was measured (linkage provenance)

Every metric below **except recommendation** uses a verified real edge (`.agents/memory/knowledge-graph-linkages.md`); no fabricated links. **Recommendation is the one exception** — it is a *lexical proxy*, not a stored edge (called out explicitly).

| Metric | Source / join | Edge type |
|---|---|---|
| Concern Count | `capadex_concerns_master.relational_bridge_tag` | real |
| Signal Count | `capadex_atomic_signals.relational_bridge_tag` (direct) | real |
| Question Count | `capadex_clarity_questions.master_bridge_tag` | real |
| Capability / Problem | `capability_problem_map.{capability,problem}_concern_id` (concern_id) | real |
| Behaviour | `behavior_library.concern_id` | real |
| Archetype | `archetype_concern_map.concern_id` → `archetype_key` | real |
| Intervention | concern → `archetype_concern_map.archetype_key` → `pil_intervention_library.archetype_key` | real (archetype-grounded) |
| Recommendation | tag ↔ `recommendation_library.anchor_construct` | **lexical proxy** (no stored edge; runtime-bound in production) |
| Overlap / Consolidation | `construct_similarity_map` (pairs + `recommended_group`) | real |

**Honest caveats**
- **Recommendation coverage is the weakest measure.** Recommendations bind to the ontology only at runtime via a 33-value construct namespace (`anchor_construct`); the per-tag figure here is a lexical approximation (exact / prefix / first-token match), not a stored edge.
- **Capability == Problem** by construction (same map row), so they are not independent signals of health.
- Signal Count counts **atomic** signals (the diagnostic units); the 20 high-level `capadex_signals` are not the per-tag measure.

## 8. Recommended next phases (for review — not executed)

1. **Signal grounding sprint** — close the 7.6% → target by mapping the ~300 signal-less bridge tags to existing atomic signals (no new signals fabricated). Highest leverage on the health score.
2. **First-class the 4 RED constructs** — give Leadership / Decision Making / Future Readiness / Entrepreneurship a dedicated bridge tag, clarity questions, and a recommendation construct.
3. **Consolidate the top similarity families** — start with Emotional (40), Decision (14), Time (7); collapse cross-tag duplicates surfaced in §5.
4. **Bridge the construct namespace to bridge tags** — author a static `bridge_tag ↔ construct` map so intervention/recommendation coverage becomes measurable (and reliable) outside runtime.
