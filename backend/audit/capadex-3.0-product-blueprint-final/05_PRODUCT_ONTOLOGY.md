# 05 · Product Ontology

ONE canonical product-ontology blueprint. Promotes the Operating-Model ontology validation (`05`) into the
frozen set. The ontology layer is CAPADEX's semantic backbone and a genuine differentiator.

## Ontology integrity (honest, FROZEN principles)
- **Abstain-never-fabricate** is platform-wide: orphan bridge tags are flagged, role-DNA abstains where coverage
  is weak, AI degrades honestly (null source tags, inert without `OPENAI_API_KEY`).
- **Coverage ⟂ Confidence ⟂ Evidence** are three independent axes, never blended.
- **Existence ≠ population:** the signal ontology auto-creates schema but still needs a manual seed; role-DNA
  coverage is partial by design — honest data-coverage gaps, not bugs.
- **Namespace collision guarded:** `pil_kg_*` (PIL knowledge graph) vs the live Employability `kg_*` graph —
  never bare `kg_*`; a documented prior hazard, now fenced.
- **Two ontologies kept deliberately separate:** LBI (`lbi_*`) ⟂ Competency (`onto_*`) — independent products,
  never bridged.

## Canonical decisions (FROZEN)
1. The **competency genome** is the authoritative skill ontology; all skill claims resolve to it.
2. **Signals are concern-diagnostic**; strengths come ONLY from CSI positive_factors / positive longitudinal
   growth — never from raw concern-signal magnitude.
3. **Outcome ontology** is canonical as a target model; its realized layer is forward work (13).
4. Platform-intelligence ontology stays **dormant** until deliberately activated (not in this phase).

## Verdict
**ONE product ontology — formal, versioned, governance-aware, honesty-engineered. FROZEN.** The only gaps are
**population/coverage** (seed signal ontology, expand role-DNA) and the **dormant** platform-intelligence
ontology — both enhancement-only, both forward work.
