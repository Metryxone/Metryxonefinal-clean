# MX-302A — Career Launchpad · Rendered-UI E2E check

## What this proves (and why it is distinct from the DB harness)
`e2e-mx302a.ts` proves the **read path** (`readEffectiveStage` → `effectiveExperience`)
resolves a returning no-profile user to Launchpad at the API level. It does NOT
exercise the rendered UI. `CareerBuilderPage.tsx` only lands the user on the
Launchpad tab when (a) the `careerLaunchpad` flag is ON, (b) `/api/career/experience/enabled`
returns ok, and (c) the `/api/career/experience` fetch succeeds and its `targetTab`
is applied in the page-load routing effect (`src/pages/CareerBuilderPage.tsx`
~lines 880-901). A transient API failure or a regression in that effect could
silently leave a no-profile user on the default `dashboard` (Command Center) tab.

This check closes that gap: it logs in through the real browser as a returning
career_seeker **with no `career_seeker_profiles` row** and asserts the landing
tab is the Launchpad tab (`fresher-hub`, labelled "Career Launchpad"), NOT
`dashboard`. It repeats the assertion for a **student-role** no-profile user
(whose stage is role-derived to `student` → Launchpad).

## Preconditions
1. Flag ON: `FF_CAREER_LAUNCHPAD=1` for the running **Backend API** workflow
   (the UI path is byte-identical-OFF without it). In the workspace this is set
   as a development env var; restart Backend API after setting it.
2. Both workflows running: `Backend API` (:8080) and `Start application` (:5000).
3. Fixtures seeded: `cd backend && npx tsx audit/mx-302a/ui-e2e-seed.ts seed`
   (creates two loginable @example.com users with NO profile row).
   Tear down with `... ui-e2e-seed.ts clean`.

## Test plan (run via the Playwright testing subagent)
```text
SCENARIO A — career_seeker, no profile row → auto-lands on Career Launchpad
1. [New Context] Create a new browser context
2. [Browser] Navigate to the login page (path: /login)
3. [Browser] Click [data-testid="role-dropdown-trigger"], then click [data-testid="role-select-job_seeker"]
4. [Browser] Fill [data-testid="input-username"] with "mx302a-ui-career@example.com"
5. [Browser] Fill [data-testid="input-password"] with "Launchpad!E2E#2026"
6. [Browser] Click [data-testid="button-login"]
7. [Verify] (on the Career Builder page, no manual tab clicks)
   - The URL contains /career-builder
   - The sidebar nav item "Career Launchpad" is the ACTIVE item (dark highlighted, white text)
   - The "Dashboard" sidebar item is NOT active
   - The main content shows the Career Launchpad header (h1 text "Career Launchpad")
     and Fresher Hub sub-tabs (Readiness, Campus Drive Tracker, Project Portfolio, Aptitude, First Job Guide)

SCENARIO B — student role, no profile row → Career Launchpad on the Career Builder
1. [New Context] Create a new browser context
2. [Browser] Navigate to the login page (path: /login)
3. [Browser] Fill [data-testid="input-username"] with "mx302a-ui-student@example.com"
4. [Browser] Fill [data-testid="input-password"] with "Launchpad!E2E#2026"
5. [Browser] Click [data-testid="button-login"]
6. [Browser] Navigate to the Career Builder (path: /career-builder)
7. [Verify]
   - The sidebar nav item "Career Launchpad" is the ACTIVE item (dark highlighted, white text)
   - The "Dashboard" sidebar item is NOT active
   - The main content shows the Career Launchpad header (h1 text "Career Launchpad")
```
