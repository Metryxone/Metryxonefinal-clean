# CAPADEX Problem Intelligence Layer

## Phase 1.5 – Concern Ontology Normalization

### Objective

The Phase 1 audit revealed that the current concern structure mixes multiple semantic entities into a single concern model.

Examples:

Capability:

* Self-Directed Learning Skills
* Career Direction Clarity
* Presentation Skills

Problem State:

* Self-Learning Deficit
* Career Direction Ambiguity
* Presentation Anxiety

The current concern architecture does not clearly distinguish between capabilities and problem states.

This phase will normalize the ontology before archetype clustering begins.

---

# CRITICAL RULE

Do not proceed to Phase 2.

Do not build archetypes.

Do not generate problem libraries.

Do not generate search intents.

Stop after completing ontology normalization and wait for approval.

---

# PHASE 1.5A – SEMANTIC RECLASSIFICATION

Reclassify all concerns using the concern name and semantic meaning rather than concern_category suffixes.

Allowed classifications:

* Capability
* Problem
* Behavior
* Trait
* Outcome
* Risk

Examples:

Capability:

* Career Direction Clarity
* Communication Skills
* Decision-Making Skills
* Academic Planning Skills

Problem:

* Career Ambiguity
* Decision Anxiety
* Communication Difficulty
* Academic Confusion

Behavior:

* Procrastination
* Avoidance
* Excessive Comparison

Trait:

* Curiosity
* Resilience
* Adaptability

Outcome:

* Burnout
* Academic Success
* Career Readiness

Risk:

* Burnout Risk
* Career Misalignment Risk

---

# PHASE 1.5B – CAPABILITY / PROBLEM SEPARATION

Identify all concerns where:

A positive capability is paired with a deficit-state category.

Examples:

| Capability                    | Problem                    |
| ----------------------------- | -------------------------- |
| Self-Directed Learning Skills | Self-Learning Deficit      |
| Career Direction Clarity      | Career Direction Ambiguity |
| Presentation Skills           | Presentation Anxiety       |
| Emotional Stability           | Emotional Instability      |

Create:

capability_problem_map

Fields:

* mapping_id
* capability_concern_id
* capability_name
* problem_concern_id
* problem_name
* confidence_score
* mapping_reason

---

# PHASE 1.5C – ONTOLOGY NORMALIZATION

Create a canonical concern model.

Every concern must belong to one of:

### Canonical Capability

Example:

Career Direction Clarity

### Canonical Problem

Example:

Career Direction Ambiguity

### Canonical Behavior

Example:

Career Avoidance

### Canonical Trait

Example:

Career Curiosity

### Canonical Outcome

Example:

Career Readiness

### Canonical Risk

Example:

Career Misalignment Risk

Create:

normalized_concern_ontology

Fields:

* ontology_id
* concern_id

* canonical_entity
* confidence_score

# PHASE 1.5D – DUPLICATE CONSTRUCT DETECTION

Detect concerns that represent the same underlying construct.

Examples:

* Career Direction Clarity
* Career Decision Confidence
* Career Choice Readiness

May belong to the same capability family.

Create:

construct_similarity_map

Fields:

* concern_a
* concern_b
* similarity_score
* recommended_group
