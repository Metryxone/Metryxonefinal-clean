BUG FIX TASK — PRAGATI CHAT FREEZES AFTER REOPENING WINDOW

ISSUE:
When a user closes the Pragati chat window/tab and reopens it, the UI shows:

"Welcome back. Let's continue from where you left off."

But:
- the input box is disabled or inaccessible
- users cannot type
- the conversation never restores
- the app appears frozen
- no recovery or fallback occurs

This is a HIGH PRIORITY production bug because it breaks trust and session continuity.

GOAL:
Make the chat ALWAYS recover safely after reopening.

REQUIREMENTS:

1. PREVENT PERMANENT LOADING STATE
- Ensure loading state always exits
- Add a fail-safe timeout (4–5 seconds max)
- Even if restore/session fetch fails, users must still be able to type

Example behavior:
- app attempts session restore
- if successful → continue previous chat
- if failed → gracefully start fresh chat
- NEVER leave UI frozen

2. FIX STUCK STREAMING STATE
Check if:
- isStreaming
- isTyping
- isLoading
- pendingResponse
or similar flags remain true after tab close/reopen.

On component mount:
- safely reset invalid streaming states
- ensure chat input becomes usable

3. SAFE LOCAL STORAGE RESTORE
Current restore logic may be failing silently.

Implement defensive restore logic:

- validate localStorage/sessionStorage data
- wrap JSON parsing in try/catch
- detect corrupted or incomplete chat state
- clear invalid state automatically

If restore fails:
- clear only broken chat state
- DO NOT crash app
- continue with fresh session

4. ADD AUTO-RECOVERY UX
Instead of frozen blank screen:

Show:
"Restoring your previous conversation..."

If restore exceeds timeout:
"We couldn't restore the previous session safely. Starting a fresh conversation."

Then automatically recover.

5. REMOVE INVISIBLE UI BLOCKERS
Check for:
- overlays
- fixed divs
- loading screens
- z-index blockers
that may intercept clicks after reopening.

Ensure input area is always interactable after loading completes.

6. ADD DEBUG LOGGING
Add temporary logs for:
- session restore start
- restore success
- restore failure
- loading timeout triggered
- localStorage parse errors
- streaming state reset

This will help diagnose future session issues.

7. IMPLEMENT FALLBACK RECOVERY
If chat restore completely fails:

Automatically:
- clear corrupted session state
- reset loading flags
- reopen clean chat safely

WITHOUT requiring manual refresh.

IMPORTANT:
Users should NEVER get trapped in a non-interactive state.

SUCCESS CRITERIA:
- closing and reopening Pragati always allows chatting again
- previous session restores safely OR falls back gracefully
- no frozen screens
- no disabled input forever
- no blank loading state
- no manual refresh required

Also:
- identify the ROOT CAUSE
- explain exactly what was causing the freeze
- summarize all fixes implemented
