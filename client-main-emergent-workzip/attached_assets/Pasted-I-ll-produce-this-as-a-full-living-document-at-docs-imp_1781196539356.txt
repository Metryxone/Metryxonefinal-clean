The 6 Phases
Phase 1 — Ontology + Assessment (Months 1–4)
The single hardest dependency is the identity bridge: users.email column + student_subscriptions.user_id FK must ship Sprint 1 or all commercial work is blocked. Unifies 3 disconnected LBI subsystems, activates the WC-3 chain in sequence (Stage → Outcome → Journey → Longitudinal), and re-labels 370 existing signal families to move signal coverage from 7.6% → 60%. Closes with Razorpay B2C launch behind a security gate. 7.5 person-months.

Phase 2 — Employability + Career Builder (Months 3–7)
O*NET bulk import (1,110 occupations, 35k+ skill links) activates the Fitment Panel and removes the Provisional flag. Peer benchmarking goes live once k ≥ 30. First enterprise B2B dashboard launched with cohort EI heatmap and CSV export. Mentor matching wired to real profiles, not the current universal fallback. 10 person-months.

Phase 3 — Learning Intelligence (Months 5–9)
Net-new module: gap engine derives learning priorities from CAPADEX constructs + EI deficits → path builder → course recommender → progress tracker → EI feedback loop. Micro-learning cards reuse the existing intervention_library (no new content authoring for v1). Static catalog of 200 courses ships day-1; Coursera/NPTEL API integration follows. 9.5 person-months.

Phase 4 — Future Readiness (Months 9–13)
Back-half pipeline already exists (outcome model rows, WC-9 seed); this phase fills the content. AI exposure scores per occupation (O*NET task ratings × automation probability), reskill vs upskill binary decision, 1/3/5-year outcome projections, and a premium downloadable report at ₹999. One shared AI-skill taxonomy bridges CAPADEX constructs ↔ occupation exposure ↔ learning paths. 10 person-months.

Phase 5 — Talent Passport (Months 11–15)
Converts the JSONB snapshot into a signed, verifiable, portable credential. Public shareable URL, controlled disclosure wizard, employer portal with anonymised talent search (k ≥ 30), Open Badge 3.0 issuance, and a public /verify/:hash endpoint. PII invariant enforced at the serialisation layer — CI fails the build if a PII field is detected in any public payload. 9.5 person-months.

Phase 6 — AI Intelligence (Months 13–18)
Multi-model router (GPT-4o / GPT-4o-mini / Claude by cost-quality tradeoff), fine-tuning pipeline on curated concern→insight pairs, Pragati moved to Redis-backed sessions for horizontal scale, WC-7 decision chain completion, A/B prompt testing framework, and predictive risk alerts. AI Governance productised as a standalone B2B SaaS offering. 12 person-months.

Critical path
Phase 1 is the root — everything depends on it. Phases 2 and 3 run in parallel from Month 3 onward. Future Readiness consumes learning gap data from Phase 3. Talent Passport aggregates Phases 1–4. AI Intelligence runs as a continuous capability layer from Month 13.
