STEP 1 — CREATE NEW V2 DATABASE MIGRATION
=========================================

Create:
backend/migrations/20260630_competency_runtime_v2.sql

Add the following new tables:

1. competency_runtime_contexts
   Purpose:
   Stores runtime user competency context.

Columns:

* id UUID PRIMARY KEY
* user_id BIGINT NOT NULL
* industry_id UUID
* function_id UUID
* sub_function_id UUID
* role_id UUID
* layer_id UUID
* complexity_model_id UUID
* geography TEXT
* org_maturity TEXT
* team_scale TEXT
* seniority_band TEXT
* assessment_mode TEXT
* runtime_version TEXT DEFAULT '2.0.0'
* created_at TIMESTAMP DEFAULT NOW()

Indexes:

* idx_crc_user
* idx_crc_role
* idx_crc_industry

---

2. role_dna_profiles_v2
   Purpose:
   Stores contextual role DNA models.

Columns:

* id UUID PRIMARY KEY
* role_id UUID NOT NULL
* industry_id UUID
* layer_id UUID
* complexity_model_id UUID
* dna_name TEXT
* dna_description TEXT
* dna_version TEXT
* readiness_model JSONB
* behavioral_model JSONB
* technical_model JSONB
* leadership_model JSONB
* strategic_model JSONB
* execution_model JSONB
* default_weightings JSONB
* expected_levels JSONB
* confidence_model JSONB
* metadata JSONB
* is_active BOOLEAN DEFAULT TRUE
* created_at TIMESTAMP DEFAULT NOW()

Indexes:

* idx_role_dna_role
* idx_role_dna_layer
* idx_role_dna_industry

---

3. competency_runtime_weights
   Purpose:
   Stores dynamically generated competency weights.

Columns:

* id UUID PRIMARY KEY
* role_dna_id UUID
* competency_id UUID
* competency_code TEXT
* importance_weight NUMERIC(5,2)
* expected_level NUMERIC(5,2)
* minimum_threshold NUMERIC(5,2)
* growth_priority NUMERIC(5,2)
* criticality TEXT
* weighting_reason TEXT
* weighting_context JSONB
* created_at TIMESTAMP DEFAULT NOW()

Indexes:

* idx_runtime_weights_role_dna
* idx_runtime_weights_competency

---

4. competency_context_modifiers
   Purpose:
   Stores contextual runtime modifiers.

Columns:

* id UUID PRIMARY KEY
* modifier_type TEXT
* modifier_name TEXT
* modifier_target TEXT
* modifier_effect JSONB
* adjustment_weight NUMERIC(5,2)
* active BOOLEAN DEFAULT TRUE
* created_at TIMESTAMP DEFAULT NOW()

Seed examples:

* Startup modifier
* Enterprise modifier
* Healthcare modifier
* AI/ML modifier
* Leadership modifier
* Executive modifier

---

5. competency_resolution_history
   Purpose:
   Stores audit trail of runtime competency resolution.

Columns:

* id UUID PRIMARY KEY
* user_id BIGINT
* runtime_context_id UUID
* resolved_role_dna_id UUID
* resolution_inputs JSONB
* resolution_outputs JSONB
* confidence_score NUMERIC(5,2)
* explainability JSONB
* created_at TIMESTAMP DEFAULT NOW()
