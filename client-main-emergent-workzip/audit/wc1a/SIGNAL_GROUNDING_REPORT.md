## 0. Question

WC-1 found signal coverage at **25 / 328 bridge tags (7.6%)** — ~300 tags have no atomic-signal grounding.
This phase asks one thing: **can the EXISTING atomic-signal pool be reused to ground those tags, or do we need to author new signals?**

## 1. Headline answer

**Yes — grounding is feasible for every signal-less tag by reusing existing signals. No tag requires new signals to reach at least partial grounding.**

The reason is structural: the atomic-signal pool is semantically **rich but mislabelled**. There are **15,972 atomic signals spanning 370 families across 20 domains**, but they are bridge-tag-labelled into only **28 mostly catch-all buckets** (STRENGTH_SIGNAL alone holds 8,970 signals across 282 families; ADJUSTMENT_COPING holds 3,286 across 312 families). The diagnostic vocabulary a tag needs almost always already exists — it is just filed under a catch-all tag instead of the construct's own tag.

| Classification | Tags | Meaning |
|---|---:|---|
| ALREADY_GROUNDED | 25 | has atomic signals today |
| **GREEN — groundable with existing signals** | **182** | a strongly-matching signal family already exists |
| **YELLOW — partial grounding** | **121** | only a moderately-matching family exists; reuse works but with reduced precision |
| **RED — requires new signals** | **0** | no existing family fits at all |

## 2. Method (and why it is honest)

1. **Family profiles** — each of the 370 signal families gets an IDF-weighted token profile from its `family_name + domain_name` and every member signal's name/label/category.
2. **Tag profiles** — each bridge tag gets a profile from its tag name + the `display_label`/`concern_cluster` of all its concerns.
3. **Match** — IDF-weighted cosine similarity, tag-vs-family. The `candidate_signal_count` that drives grounding = the members of families **at or above the YELLOW floor** only. The relationship path is explicit: `tag → domain → family → N atomic signals`. (`candidate_signal_reuse_map.csv` additionally exports the top-3 families per tag for analyst context — including any below-floor third family — but below-floor families do **not** count toward the reusable candidate total or the classification.)
4. **Calibration against ground truth** — thresholds are NOT hand-picked. The 25 already-grounded tags are run through the SAME matcher; their best-family similarity (p25 = **0.2139**) defines the GREEN threshold `TAU_HIGH = 0.2139`. The YELLOW floor is `TAU_LOW = 0.0963` (45% of GREEN). GREEN also requires ≥3 candidate signals.

**No signals are fabricated.** Every candidate is an existing `capadex_atomic_signals` row. The output names the exact family and example signals so each suggestion is auditable.

### Why RED = 0 is a real finding, not a loose threshold
RED = 0 means **no signal-less tag falls below the calibrated YELLOW floor under this thresholded lexical-family matcher** — it is not an absolute semantic proof. The best-family similarity of signal-less tags ranges **0.109 (min) → 0.229 (median) → 0.417 (max)**. The weakest matches were spot-checked and remain topically plausible (e.g. `METACOGNITIVE_DEVELOPMENT → reflective_learning_signals`, `ADAPTABILITY_COACHING → adaptability_behavior_signals`, `STUDENT_RECOVERY → evaluation_anxiety_signals`). The finding is robust to a stricter floor:

| YELLOW floor | source | tags that would become RED |
|---|---|---:|
| 0.0963 | calibrated (`TAU_HIGH × 0.45`) | 0 |
| 0.1279 | grounded-tags' **minimum** best-family sim | 1 (ADAPTIVE_LEADERSHIP, 0.109) |
| 0.1733 | grounded-tags' **p10** best-family sim | ~18 |

So even at the strictest defensible floor (the worst real grounded tag), only a single tag flips to RED. **Reuse feasibility is therefore near-universal; the real discriminator is grounding QUALITY (GREEN strong vs YELLOW partial), not feasibility.** The 121 YELLOW tags can be reused today but are the honest candidates for dedicated signal authoring later, where precision matters.

## 6. FINAL RECOMMENDATION

- **GREEN (groundable using existing signals): 182 tags + 9 of 10 priority constructs.** Re-link, in a future remediation phase, the best-matching family's atomic signals to these tags. No authoring required.
- **YELLOW (partial grounding): 121 tags + Entrepreneurship.** Reuse is possible today but precision is limited; these are the honest candidates for **dedicated signal authoring later**, where construct fidelity matters most.
- **RED (requires new signals): 0.** The existing signal pool semantically covers the whole bridge-tag space.

**The systemic risk WC-1 flagged (7.6% signal coverage) is a labelling/wiring problem, not a missing-content problem.** The actionable single biggest lever is re-pointing existing atomic signals out of the catch-all tags (STRENGTH_SIGNAL / ADJUSTMENT_COPING) onto the construct tags they semantically belong to.
