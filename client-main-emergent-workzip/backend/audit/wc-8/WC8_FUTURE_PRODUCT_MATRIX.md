Which future-readiness products should exist, scored on the WC-8 dimensions. Build difficulty is
judged against **what already exists** (outcome-model + journey-route + bridge precedents are cheap;
net-new reference data is expensive).

## G.2 Why these scores (grounded)
- **Future Employability Index (P1, Low):** `employability_readiness` outcome + `employability_index`
  journey are both **live**. This is a reframe/upgrade, not a build — highest leverage.
- **AI Career Navigator (P1, Medium):** 440 AI-context Qs already exist (demand proven), but it needs
  an AI-resilient skill taxonomy + an `ai_readiness` outcome model. The outcome model is cheap
  (7 precedents); the taxonomy is the real cost.
- **Future Skills Planner (P2, Medium):** the WC-7B growth-plan bridge already produces roadmaps;
  this consumes the same bridge + the AI/future-skill taxonomy. Shares Track B's reference data.
- **Career Resilience Index (P2, Low):** adaptability/resilience is the single largest theme
  (395 concerns / 1,480 Qs) — a resilience outcome could be composed cheaply from existing content.
- **Entrepreneurship Index (P3, High):** the only product that needs **content seeding first**
  (89 context Qs, 6 concerns) — defer.

## G.3 The dependency that shapes the build order
Three P1/P2 products (AI Navigator, Future Skills Planner, and the AI-resilience lens on
Employability 2.0) **share one missing asset: an AI-resilient / future-skill taxonomy + an
occupation-exposure reference.** Build that asset once and three products unlock. That single shared
dependency is the highest-leverage item in all of WC-8.

## G.4 Recommended sequence
1. **Future Employability Index 2.0** (compose-only, immediate).
2. **AI-resilient skill taxonomy** (shared reference data — the keystone).
3. **AI Career Navigator** + **Future Skills Planner** (both consume #2).
4. **Career Resilience Index** (compose from existing adaptability corpus).
5. **Entrepreneurship Index** (only after content seeding).
